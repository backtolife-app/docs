# Schema Reference

Every table in the backend, grouped by domain. Column types follow Laravel's schema builder (`$table->id()` → `bigint unsigned auto_increment`, `$table->foreignId(...)` → `bigint unsigned` with FK constraint, etc.).

Unless noted, each table has `created_at` + `updated_at` (`timestamps()`).

---

## Identity & auth

### `users`

The single user table for every role — Admin, standard User (child), and Parent are differentiated by the `role` column. Email is globally unique; the `unique_email_per_role_in_users_table` migration (2026-03-24) relaxed this to allow a user to register once per role (e.g. parent + child on the same address).

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| name | string nullable | |
| email | string nullable (unique, role-scoped) | |
| email_verified_at | timestamp nullable | |
| password | string nullable | Bcrypt. |
| avatar | text nullable | URL. |
| terms | boolean default `false` | T&C accepted. |
| is_admin_created | boolean default `false` | |
| device_id | string nullable | Originally used for device pinning. |
| language | string nullable | `en` \| `es` \| … |
| provider_id | string nullable (indexed) | Social-auth provider (`google`, `apple`). |
| provider_user_id | string nullable (indexed) | External user id from the provider. |
| google_id | string nullable | Added by `add_google_id_in_users_table`. |
| reset_code | string nullable | OTP for password reset. |
| reset_code_expires_at | timestamp nullable | |
| role | enum (`Admin`, `User`, `Parent`) default `User` | |
| status | enum (`Pending`, `Rejected`, `Active`, `Deactivate`, `Banned`) default `Active` | |
| battery_level | unsigned tinyint nullable | *(Added 2026-04-17)* 0–100. Reported by the child app via `PUT /battery`. |
| battery_charging | boolean nullable | *(Added 2026-04-17)* Whether the device is plugged in. |
| battery_updated_at | timestamp nullable | *(Added 2026-04-17)* Last battery report time. |
| remember_token | string | |
| timestamps + softDeletes | | |

### `personal_access_tokens`

Laravel Sanctum's token store — driven by the `laravel/sanctum` package default migration.

### `password_reset_tokens`, `sessions`

Laravel defaults. `sessions` is only used if session-driver-based auth is enabled on the admin panel.

## Parental controls

See [Parental Controls Domain](parental-controls.md) for the conceptual guide.

### `family_links`

Links a parent user to a child user. Many-to-many by design — a child may have multiple parents (shared custody), a parent may link many children (up to 10 per business rule).

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| parent_user_id | FK → `users.id` nullable, cascade delete | |
| child_user_id | FK → `users.id` nullable, cascade delete | |
| status | enum (`pending`, `active`, `revoked`) default `pending` | |
| invite_code | string unique nullable | 8-char code generated by the child's `/child/invite` endpoint. |
| invite_expires_at | timestamp nullable | Typically 24 h. |
| timestamps | | |

### `parental_controls`

One row per active `family_link`. Holds the parent's configured restrictions for that child.

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| family_link_id | FK → `family_links.id` cascade delete | Effectively 1:1. |
| settings_locked | boolean default `false` | When true, child cannot edit schedule/activities in their own app. |
| daily_screen_time_limit_minutes | int nullable | **Legacy single-value column**, kept for back-compat. |
| daily_limit_weekday_minutes | int nullable | *(Added 2026-04-16)* Mon–Fri daily cap. |
| daily_limit_weekend_minutes | int nullable | *(Added 2026-04-16)* Sat–Sun daily cap. |
| always_allowed_apps | json nullable | *(Added 2026-04-16)* Array of `{ name, emoji?, package? }`. Apps that stay unlocked during sleep windows. |
| notification_preferences | json nullable | *(Added 2026-04-17)* Per-child notification toggles + quiet hours. Shape: `{ screen_time_exhausted, blocked_app_attempt, extra_time_request, sleep_reminder, daily_summary, child_message, quiet_hours_enabled, quiet_hours_start, quiet_hours_end }` |
| timestamps | | |

### `sleep_schedules` *(added 2026-04-16)*

Device-wide "bedtime lock" windows. Up to two rows per child: one for weeknights, one for weekend nights. Enforced by the child app (planned — see [PLAN.md](../../../parent/PLAN.md)).

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| family_link_id | FK → `family_links.id` cascade delete | |
| bucket | string (`weeknight` \| `weekend`) | |
| enabled | boolean default `true` | |
| start_time | time | `HH:MM:SS` in the stored timezone. |
| end_time | time | `HH:MM:SS`. If `< start_time` the window crosses midnight. |
| days | json | Array of ISO-8601 weekday ints (1 = Mon … 7 = Sun). |
| timezone | string nullable | IANA name, e.g. `Europe/Rome`. |
| timestamps | | |
| **unique** | `(family_link_id, bucket)` | Enforces max 2 rows per child. |

## Activity & usage

### `screen_time_daily` *(2026-03-31)*

Per-app, per-day totals reported by the child app. This is what the parent dashboard bar chart reads. Replaces the older `screen_usages` table.

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| user_id | FK → `users.id` cascade delete | Child user. |
| date | date | Local date in the child's timezone (client-supplied). |
| timezone | string(64) | IANA name. |
| app | string(32) | Currently `youtube` \| `instagram`. |
| source | string(16) | `webview` (tracked inside BackToLife) \| `native` (tracked via UsageStats). |
| seconds | unsigned int default 0 | Day's total. |
| updated_at | timestamp | Auto-refreshed via `useCurrent()->useCurrentOnUpdate()`. No `created_at`. |
| **unique** | `(user_id, date, app, source)` | Upsert key. |
| **index** | `(user_id, date)` | Date-range queries. |

| logout_time | timestamp nullable |
| total_time | string nullable (HH:MM:SS) |
| timestamps | |

### `strict_shedules`

Per-child social-media filter. **Named `strict_shedules` (missing c) intentionally — do not "fix" the typo without a rename migration.** Holds which YouTube / Instagram features to hide and in what time window. One row per child via upsert on `user_id`.

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| user_id | FK → `users.id` cascade delete nullable | Child user id. |
| device_id | string nullable | |
| timezone | string nullable | |
| youtube_shorts | boolean nullable | |
| youtube_comments | boolean nullable | |
| instagram_reels | boolean nullable | |
| instagram_explore | boolean nullable | |
| instagram_likeComments | boolean nullable | |
| instagram_story | boolean nullable | |
| instagramDmOnly | boolean nullable | |
| start_time | time nullable | Window start (server TZ). |
| end_time | time nullable | Window end (may be next day). |
| days | json nullable | Days the filter applies. |
| is_active | boolean nullable | |
| last_start_notified_date | date nullable | De-dupes push notifications. |
| last_end_notified_date | date nullable | |
| timestamps | | |

### `strict_data`

Session-level audit trail of when each strict schedule became active / was disabled. Used to compute enforcement streaks.

| Column | Type |
|---|---|
| id | bigint PK |
| strict_id | FK → `strict_shedules.id` cascade delete |
| started_at | datetime nullable |
| disabled_at | datetime nullable |
| timestamps | |

### `screen_usages` *(legacy)*

Pre-`screen_time_daily` usage log — per-session rather than per-day.

| Column | Type |
|---|---|
| id | bigint PK |
| user_id | FK → `users.id` cascade delete nullable |
| device_id | string nullable |
| screen_name | string nullable |
| start_time | datetime nullable |
| end_time | datetime nullable |
| timestamps | |

!!! note
    New code should write to `screen_time_daily`. `screen_usages` is kept for historical data and the activity-history endpoint.

### `app_statuses`

Per-child default state of the social-media filter toggles. Mirrors the seven boolean columns on `strict_shedules` without the time window — effectively the "what's toggled right now" snapshot independent of the schedule.

## Push & notifications

### `firebase_tokens`

FCM device tokens used to push alerts to children or parents.

| Column | Type | Notes |
|---|---|---|
| id | bigint PK | |
| user_id | FK → `users.id` (restrict on delete, cascade on update) nullable | |
| token | longtext not null | FCM registration token. |
| device_id | longtext nullable | |
| status | enum (`Active`, `Inactive`) default `Active` | |
| created_at / updated_at | timestamp auto | `useCurrent()` + `useCurrentOnUpdate()` |
| softDeletes | | |

!!! warning
    The send-side service (parent → child push commands) is **not yet built**. Only token CRUD endpoints exist today.

### `notifications`

Laravel's default polymorphic notifications table (`notifiable_type`, `notifiable_id`). Currently used for in-app alerts only.
